# Agent-Reach：联网获取内容

获取互联网公开内容 — 搜索网页、阅读任意网页、搜索 GitHub 项目。

## 何时调用

符合以下任一条件即触发：
- 用户给出一个 URL（任意网页、小红书、微博、推特、抖音、B站、V2EX、Reddit、LinkedIn、雪球、微信公众号文章、YouTube、小宇宙、播客、GitHub repo）
- 用户要求"搜一下""查一下""找一找"（含中文/英文搜索词）
- 用户说"联网""上网""看网页""帮我看""打开这个链接"
- 用户提到具体平台（关键词见下表）

## 工作流（Web Agent 环境）

### Step 1: 根据意图选择工具

| 用户意图 | 使用工具 | 参数 |
|---------|----------|------|
| 给 URL，读内容 | `web_fetch` | `url=URL` |
| 搜索一般内容 | `web_search` | `query=关键词` |
| 搜索 GitHub 项目 | `search_skill` | `query=关键词` |

### Step 2: 执行工具 → 获取内容

调用对应的工具，等待结果返回。

### Step 3: 处理结果

- 把工具返回的内容整理后直接呈现给用户
- 不要提示安装 CLI、不要提 `/tmp/` 或文件系统操作 — 这些在 Web 环境不可用
- 结果太长时适当总结，保持清晰

## 失败处理

| 情况 | 应对 |
|------|------|
| `web_fetch` 抓取失败 | 告诉用户"当前网页无法访问，请检查 URL 是否正确或稍后重试" |
| `web_search` 返回空结果 | 告诉用户"没有找到相关内容，请尝试调整搜索词" |

## 示例

用户："帮我找一下 B 站 UP 主鱼皮的最新内容"

你应该：
1. 确认匹配 B 站 + 搜索 → 调用 `web_search(query="B 站 UP 主 鱼皮 最新视频")`
2. 获取搜索结果后，整理呈现给用户
