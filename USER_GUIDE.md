# MyKB 使用文档

MyKB 是一个个人知识库管理系统，支持文章管理、AI 对话、网页导入等功能。

## 一、快速开始

### 1. 访问系统
打开网站，未登录状态下默认进入公开列表，可浏览所有公开分享的知识条目。

### 2. 注册账号
1. 点击右上角「登录」按钮
2. 切换到「注册」模式
3. 填写邮箱、密码
4. 拖动滑块完成验证
5. 点击「发送验证码」，查收邮箱并输入 6 位验证码
6. 点击「注册」即可自动登录

### 3. 登录
1. 点击右上角「登录」按钮
2. 输入账号（邮箱）和密码
3. 拖动滑块完成验证
4. 点击「登录」

**管理员账号**：账号 `admin`，密码 `admin710`

### 4. 公开 vs 私有
- **未登录用户**：只能查看公开列表，私有列表按钮灰色不可点击
- **登录用户**：可查看自己的私有记录和所有公开记录
- 导入的文章默认为私有，如需公开请在导入后修改可见性

## 二、记录管理

### 创建记录
1. 点击导航栏的「新建」按钮
2. 填写标题、分类、日期、摘要
3. 编辑内容（支持 Markdown 格式）
4. 点击保存

### 编辑/删除
- 在记录列表中点击对应记录进入详情页
- 详情页底部有编辑和删除按钮

### 分类管理
- 系统内置默认分类（前端、后端、AI/ML、阅读、DevOps、设计）
- 可在设置中自定义分类
- 公开分类由管理员维护，普通用户无法篡改

## 三、网页导入

### 导入单篇文章
1. 点击导航栏的「导入」按钮（需登录）
2. 输入文章 URL
3. 系统抓取并预览文章内容
4. 选择分类，确认后点击导入
5. 导入的文章默认为私有

### RSS 批量导入
1. 在导入对话框中切换到 RSS 模式
2. 输入 RSS 源地址
3. 勾选需要导入的文章
4. 点击批量导入

### 导入质量校验
系统会自动拦截以下异常页面，不会导入废文章：
- 反爬验证页面（CAPTCHA、环境异常等）
- 内容过短的页面（少于 80 字符）
- 常见错误页标题（403、Access Denied 等）

## 四、AI 对话

### 配置 AI
1. 点击左下角齿轮图标打开设置
2. 在 Agent 标签页配置：
   - LLM API Key
   - 模型名称（如 `deepseek-chat`）
   - API Base URL（如 `https://api.deepseek.com/v1`）
3. 可选配置 Embedding API（用于语义搜索）

### 使用对话
1. 点击侧边栏打开 AI 对话面板
2. 直接输入问题发送
3. 或使用 `/` 触发技能快捷菜单（搜索、抓取网页等）
4. AI 会基于你的知识库内容回答问题

### 对话功能
- **知识库搜索**：AI 自动搜索你的记录作为上下文
- **网页搜索**：通过 `/search` 快捷搜索网络
- **网页抓取**：通过 `/fetch` 抓取指定 URL 内容
- **多会话管理**：可创建多个对话会话，支持重命名和删除

## 五、搜索

- 在列表页顶部搜索框输入关键词
- 搜索范围：当前列表模式（公开/私有）下的记录
- 支持标题、摘要、内容全文搜索

## 六、设置

### 通用设置
- 语言切换（中文/英文）
- 主题模式

### Agent 设置
- LLM 配置（API Key、模型、Base URL）
- Embedding 配置（用于语义搜索）
- 最大 Token 数

### 存储设置
- Turso 数据库配置
- 七牛云 R2 配置（图片上传）

## 七、Vercel 部署须知

### 必须配置的环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `JWT_SECRET` | JWT 签名密钥（≥32字符，**必须替换为强随机值**） | `my-super-secret-key-xxx` |
| `TURSO_DB_URL` | Turso 数据库 URL | `libsql://my-db.turso.io` |
| `TURSO_DB_TOKEN` | Turso 数据库 Token | `eyJhbGci...` |
| `SMTP_USER` | QQ邮箱地址 | `123456789@qq.com` |
| `SMTP_PASS` | QQ邮箱授权码（非密码） | `xxxxxxxxxxxxx` |

### 可选环境变量

| 变量名 | 说明 |
|--------|------|
| `ADMIN_PASSWORD_HASH` | 管理员密码的 bcrypt hash |
| `SMTP_HOST` | SMTP 服务器地址（默认 `smtp.qq.com`） |
| `SMTP_PORT` | SMTP 端口（默认 `465`） |
| `EMAIL_FROM` | 发件人显示名称（默认使用 SMTP_USER） |
| `QINIU_R2_ACCESS_KEY` | 七牛云 R2 Access Key |
| `QINIU_R2_SECRET_KEY` | 七牛云 R2 Secret Key |
| `QINIU_R2_BUCKET` | 七牛云 R2 存储空间名 |
| `QINIU_R2_DOMAIN` | 七牛云 R2 自定义域名 |

### Vercel 部署步骤
1. 在 Vercel 导入 GitHub 仓库
2. 在 Vercel Dashboard → Settings → Environment Variables 配置上述变量
3. **确保 JWT_SECRET 不是测试密钥**
4. 点击 Deploy
5. 部署完成后，首次访问会自动初始化数据库表结构

### 已知限制
- 向量索引使用内存模式（Serverless 不支持本地文件持久化），冷启动时需重建索引
- 对话历史仅存储在浏览器 localStorage，不跨设备同步
- 滑块验证码为前端验证，主要起防机器人骚扰作用
