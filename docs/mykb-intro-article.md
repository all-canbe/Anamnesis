# 知忆 — 个人知识记录系统

## 1. 创意名称 + 创意介绍

**想解决什么问题：**
日常学习过程中，我们在浏览器、终端、手机等多个场景下获取知识碎片，却缺少一个统一、轻量、可随时记录和回顾的个人知识库。笔记散落在各种工具中，检索困难，更谈不上系统化管理。更关键的是，大多数笔记工具只有"存"没有"用"——知识被存储后缺乏智能检索和关联发现的能力。

**为什么会想到做这个：**
我日常需要记录大量技术笔记（前端、后端、AI、阅读心得等），尝试过 Notion、Obsidian 等工具，但它们要么太重、要么数据锁定在特定格式。我想要一个**文件即数据**、**Git 友好**、**CLI + Web 双通道**的极简知识管理系统——写笔记就像写代码一样自然。同时，随着阿里通义实验室开源了 Zvec（嵌入式向量数据库，号称"向量数据库界的 SQLite"），我看到了将传统 Markdown 文件存储与向量语义检索结合的可能性——**MD 驱动内容存储，Zvec 驱动语义检索**。

**大概是什么产品：**
一个基于 Next.js 的个人知识记录 Web 应用 + 配套 CLI 命令行工具，采用 **MD + Zvec 双驱动架构**：
- **MD 驱动**：知识以 Markdown 文件存储在本地或 GitHub 仓库，Git 版本控制，零数据库依赖
- **Zvec 驱动**：阿里通义实验室开源的嵌入式向量数据库，为知识库提供语义检索、相似推荐、RAG 问答能力
- **Turso 主存储**：SQLite 兼容的 serverless 数据库，Web 和 CLI 直连，毫秒级响应
- 支持分类浏览、全文检索、远程管理，未来集成 AI Agent 助手进行知识探索与总结

## 2. 目标用户及痛点

**面向哪些用户：**
- 技术写作者、开发者、知识工作者
- 习惯用 Markdown 做笔记的人
- 追求数据自主权、不想被封闭生态绑定的用户
- 希望知识库具备"智能"能力（语义搜索、知识关联、AI 问答）的用户

**在什么场景下使用：**
- 日常学习时快速记录技术要点
- 通过终端 CLI 一键发布笔记（无需打开浏览器）
- 在 Web 端浏览、搜索、管理知识库
- 部署到 Vercel 后，随时随地通过公网访问
- 通过 Zvec 向量检索进行语义搜索，找到"相关但关键词不同"的知识
- 通过右侧 AI Agent 侧边栏进行知识问答、总结、探索

**当前痛点：**
- 笔记散落在多个平台，缺乏统一管理
- 传统笔记工具不支持 CLI 操作，自动化困难
- 数据格式不开放，迁移成本高
- 缺乏从"记录"到"整理"到"产出"的完整工作流
- 关键词搜索无法理解语义，找不到"相关但不包含关键词"的内容
- 知识之间缺乏关联，无法发现隐藏的知识脉络

## 3. 价值与意义

**效率提升：**
- **MD + Zvec 双驱动**：Markdown 文件存储保证数据可移植，Zvec 向量检索提供语义理解能力
- **Turso 主存储**：SQLite 兼容，HTTP 协议直连，Web 和 CLI 均直连数据库，无需经过应用层中转
- **CLI + Web 双通道**：终端发布、Web 管理，覆盖不同使用习惯
- **设置页切换服务器**：Web 端可随时切换 Turso 云服务或自部署 sqld 服务器
- **文件即数据**：Markdown 文件直接存储，Git 版本控制天然支持，无数据库依赖
- **零配置部署**：一键部署到 Vercel，环境变量配置即可连接 Turso

**商业/社会价值：**
- 数据主权回归用户——所有内容以标准 Markdown 存储，随时可迁移
- 开源友好的架构设计，可扩展为团队知识库或博客系统
- 为"个人知识管理 + AI"提供基础平台，Zvec 嵌入式向量数据库无需独立部署
- 未来可接入 Agent 进行自动总结、知识图谱构建

## 4. 技术架构亮点

- **Next.js 16 App Router** — Server Components + Server Actions，数据直读 Turso
- **MD + Zvec 双驱动** — Markdown 文件负责内容存储和版本控制，Zvec 负责向量化索引和语义检索
- **Turso 主存储** — SQLite 兼容，HTTP 协议，serverless 友好，Web/CLI 直连
- **GitHub 备份** — 定时/手动同步到仓库，保留 Git 版本历史
- **中英文双语** — React Context 实现运行时语言切换
- **CLI 工具** — 全局 npm 包 `kb`，支持 publish/list/edit/delete/status 等命令
- **设置页** — Web 端配置 Turso/sqld 服务器地址，支持随时切换
- **Agent 侧边栏** — 右侧抽屉式 AI 助手，支持知识管理/产出/探索/总结
