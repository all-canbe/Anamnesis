帮我总结一下 RSC 这篇文章的核心要点
React Server Components 核心要点：

1. 服务端渲染 — 组件在服务端运行，直接访问数据库和文件系统，无需 API 层

2. 组件分类 — Server Components 处理数据和渲染，Client Components 处理交互和状态

3. 流式传输 — 支持边渲染边发送，结合 Suspense 分块加载

4. 限制 — Server Components 不能使用 useState/useEffect，不能绑定事件

核心功能

CLI + Web 双通道知识管理

终端发布、Web 管理、远程同步、AI 增强——覆盖所有使用场景。

🌐

Web 知识库

分类浏览、分页、详情渲染（MD/HTML）、Tags 管理、中英文切换。基于 Next.js 16 App Router，Server Components 直读 Turso。

⌨️

CLI 命令行

全局 npm 包 kb，6 个命令覆盖发布/列表/编辑/删除/状态/初始化。远程模式直连 Turso，毫秒级响应。

📝

编排编辑器

内置 Markdown 编辑器 + 工具栏，支持 MD ↔ HTML 格式切换，实时预览，支持图片上传。

🗄️

Turso 主存储

SQLite 兼容，HTTP 协议直连，ACID 事务保障。Web 和 CLI 均直连，无需经过应用层中转。支持切换自部署 sqld。

🌍

中英文双语

React Context 运行时语言切换，覆盖导航、分类、详情、编辑器、设置页等全部 UI 文本。

⚙️

设置页切换服务器

Web 端设置页可配置 Turso 地址和 token，支持随时切换 Turso 云服务或自部署 sqld 服务器。

☰

三栏布局

左侧隐藏面板（文件树/视图切换/快速筛选），中间主内容区，右侧固定 AI Agent 面板。点击 ☰ 按钮呼出左侧面板。

📂

上传文件夹

支持上传整个文件夹（如个人 Skill），自动解析文件夹结构为分类和知识记录。文件夹名→分类，.md 文件→记录。

三层架构

Turso 主存储 + GitHub 备份 + Zvec 语义检索

Web 和 CLI 直连 Turso，不再经过应用层中转。GitHub 退化为备份层，Zvec 提供向量检索能力。
